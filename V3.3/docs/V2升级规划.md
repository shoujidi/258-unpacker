# V2 多层嵌套压缩解压工具升级规划

## 目标定位
多层嵌套压缩文件解压工具。

## 最大层数
最多自动处理 11 层。

## 支持格式
ZIP、7Z、TAR、GZ、TGZ、BZ2、XZ、无扩展名、错误扩展名、常见分卷。

## 解压引擎
普通 ZIP：Python zipfile 优先，失败后 7z。
7Z / TAR / GZ / TGZ / BZ2 / XZ：7z。
分卷压缩包：7z。
MP4 内嵌 ZIP：保留 detector 判断，使用 embedded_zip_offset 提取 ZIP 有效段到 cache，再用 Python zipfile 解压。

## MP4 内嵌 ZIP 特殊规则
不是简单判断文件头是否为 ZIP。
某些文件开头是正常 MP4，ZIP 数据藏在文件中间或尾部。
当密码池包含“孙笑川”时，遇到 .mp4 文件要高优先级分块扫描 PK\x03\x04。
发现 PK\x03\x04 后，不改名原 MP4；从 embedded_zip_offset 开始提取 ZIP 有效段到 cache。
使用 Python zipfile 解压临时 ZIP。
成功后删除临时 ZIP；失败后保留临时 ZIP 便于人工检查。

## 密码池规则
密码池不排序。
保持用户输入顺序。
“孙笑川”只用于判断是否高优先级扫描 MP4 内嵌 ZIP，不用于调整密码顺序。

## 7z 集成
建议内置 tools\7z\7z.exe 和 tools\7z\7z.dll。
PyInstaller 打包时需要 add-binary。

## 开发阶段
1. 文件识别器。
2. 7z 引擎。
3. ZIP 兜底到 7z。
4. MP4 内嵌 ZIP 特殊逻辑。
5. 11 层递归整合。
6. GUI 文案和打包脚本更新。
7. 轻量验收与发布清理。

## 阶段 1：文件识别器
新增 archive_detector.py。
支持识别 ZIP、7Z、GZ、BZ2、XZ、TAR、TGZ、分卷、真 MP4、MP4 内嵌 ZIP。
MP4 内嵌 ZIP 采用分块扫描 PK\x03\x04，不再只判断文件头。

## 阶段 2：7z 引擎
新增 archive_7z_engine.py。
支持查找内置 tools\7z\7z.exe。
支持 7z 密码池解压。
密码池不排序，先尝试无密码，再按用户输入顺序尝试密码。
本阶段未接入主递归流程。

## 阶段 3：ZIP 7z 兜底
ZIP 候选文件优先使用 Python zipfile 解压。
Python zipfile 失败后使用 archive_7z_engine.py 进行 7z 兜底。
支持扩展名错误或无扩展名但文件头为 ZIP 的文件。
非 MP4 ZIP 不使用 MP4 特殊路径。
本阶段未接入 7Z / TAR / GZ / TGZ / BZ2 / XZ / 分卷主流程。

## 阶段 4：7Z / TAR / GZ / TGZ / BZ2 / XZ 接入主流程
已将 7Z、TAR、GZ、TGZ、BZ2、XZ 等格式接入主流程。
这些格式统一使用内置 7z.exe / 7z.dll。
非 MP4 文件不使用 MP4 特殊路径。
MP4 内嵌 ZIP 特殊逻辑仍未在本阶段接入。
分卷压缩包正式接入留到后续阶段。

## 阶段 5：分卷压缩包接入主流程
已支持常见分卷压缩包识别与主流程解压：

- .7z.001 / .7z.002
- .zip.001 / .zip.002
- .001 / .002
- .z01 + .zip

规则：
只处理入口卷。
非入口卷直接跳过。
分卷统一使用 7z 解压。
不使用 MP4 特殊路径。
不自动删除源分卷。

## 阶段 6：MP4 内嵌 ZIP 特殊逻辑
MP4 特殊文件不是简单的“文件头就是 ZIP”。
部分文件开头是正常 MP4 容器，ZIP 数据藏在文件中间或尾部，PK\x03\x04 位置不固定。

处理规则：
- 密码池包含“孙笑川”时，高优先级扫描 MP4。
- 发现 PK\x03\x04 后，判断为 MP4 内嵌 ZIP 候选。
- 不再调用旧外部处理路径。
- 保留 detector 判断逻辑，使用 embedded_zip_offset。
- 从 offset 开始复制 ZIP 有效段到 cache。
- 使用 Python zipfile 解压。
- 成功后删除临时 ZIP。
- 失败后保留临时 ZIP。
- 原 MP4 不改名、不删除、不裁切。
- 非 MP4 文件不使用 MP4 特殊路径。

## 阶段 7：11 层递归整合
已将 ZIP、7Z、TAR、GZ、TGZ、BZ2、XZ、分卷和 MP4 内嵌 ZIP 串入统一多层递归流程。

最大递归层数：11 层。

规则：
- 普通 ZIP：Python zipfile 优先，失败后 7z。
- 7Z / TAR / GZ / TGZ / BZ2 / XZ：7z。
- 分卷：只处理入口卷，使用 7z。
- MP4 内嵌 ZIP：使用 offset 提取 ZIP 有效段到 cache，再用 Python zipfile 解压。
- 非 MP4 文件不使用 MP4 特殊路径。
- 密码池不排序。
- 不写系统 Temp。

## 阶段 9：轻量验收与发布清理

为减少 SSD 写入，本阶段不执行全量解压测试。
只做语法检查、导入检查、资源文件检查、7z 路径检查和发布目录检查。
完整解压测试仅在功能变更后手动执行，不作为每次发布前固定流程。
